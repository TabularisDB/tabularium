## What is Tabularium?

A directory of plugins (or any release-shipping artefact) with a web UI, an OAuth-driven submission flow, an OpenAPI surface, and a built-in CMS. Authors connect their GitHub / GitLab / Gitea account, pick a repo, and Tabularium installs the release webhook for them — every new tag becomes a release in the registry.

## Features

- 🧩 **Multi-provider submission** — GitHub, GitLab, Gitea (any instance)
- 🪄 **Install wizard** — structured DB form (host/port/user/pw + test-connection), bootstrap admin password printed to logs, auto-redirect into `/admin` already signed-in after restart
- 📝 **Built-in CMS** — markdown pages with widgets, translations per locale
- 🏷 **Plugin kinds** — admin-defined taxonomy (Themes, Snippets, SQL Templates, …) consumed by end-user apps as filter chips
- 🎨 **Branding** — name, colours, logo, favicon, analytics, robots policy
- 🌍 **6 UI languages** — English, Deutsch, Español, Français, Italiano, 中文 — admin-configurable
- 🗄 **Multi-dialect** — SQLite, Postgres, or MySQL (auto-detected from `DATABASE_URL`)
- 🚦 **Feature toggles** — disable submissions or requests without a redeploy
- 🪵 **Audit log** — every admin mutation tracked with actor + IP + target

## Quick start

The fastest way to run Tabularium is via the published Docker image:

```bash
curl -O https://codeberg.org/Tabularium/Tabularium/raw/branch/main/docker-compose.example.yml
mv docker-compose.example.yml docker-compose.yml

# Generate real secrets
openssl rand -hex 32  # use for JWT_SECRET
openssl rand -hex 32  # use for TOKEN_ENC_KEY

# Edit docker-compose.yml — replace both secrets above
docker compose up -d
```

Open `http://localhost:3000`. The install wizard prints a bootstrap password to the container logs:

```bash
docker compose logs registry | grep -A2 "FIRST-RUN SETUP"
```

Walk through the wizard, point it at the embedded SQLite path (or swap to Postgres / MySQL via `DATABASE_URL`), and you're done.

### Other dialects

The minimal example uses SQLite. To use Postgres or MySQL, change `DATABASE_URL` in the compose env block:

- Postgres: `postgres://user:pass@host:5432/db`
- MySQL: `mysql://user:pass@host:3306/db`

### Developing locally

If you want to hack on Tabularium itself rather than just deploy it:

```bash
git clone https://codeberg.org/Tabularium/Tabularium
cd Tabularium
bun install
docker compose -f compose.dev.yml up -d   # postgres + dragonfly

# in one terminal
cd apps/api && bun --hot src/index.ts

# in another
cd apps/frontend && bun dev
```

The bootstrap password is printed on first boot:

## Stack

| Layer | Tech |
|-------|------|
| Runtime | [Bun](https://bun.sh) (≥ 1.3) |
| HTTP | [Elysia](https://elysiajs.com) + TypeBox |
| DB | [Drizzle ORM](https://orm.drizzle.team) — sqlite / postgres-js / mysql2 |
| Cache | `Bun.redis` (Dragonfly-compatible) or in-memory |
| Frontend | SvelteKit (SPA) + [Eden Treaty](https://elysiajs.com/eden/treaty/overview) end-to-end types |
| i18n | [Paraglide JS](https://inlang.com/m/gerre34r/library-inlang-paraglideJs) |
| Docs | [Docsify](https://docsify.js.org) — no build step |
